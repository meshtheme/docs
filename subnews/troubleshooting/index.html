Before You Start

First Steps for Any Issue

Step 1: Clear Browser Cache

  • Windows: Press Ctrl + Shift + Delete
  • Mac: Press Cmd + Shift + Delete
  • Or check your browser settings

Step 2: Check WordPress Version

  1. Log in to dashboard
  2. Check bottom right for version
  3. Must be 5.8 or higher
  4. Update if needed

Step 3: Check PHP Version

  1. Go to Tools ‚Üí Site Health
  2. Look for PHP version
  3. Must be 7.4 or higher
  4. Contact hosting if too old

Step 4: Enable Debug Mode

  1. Edit wp-config.php
  2. Find define('WP_DEBUG', false);
  3. Change to define('WP_DEBUG', true);
  4. Add: define('WP_DEBUG_LOG', true);
  5. Check /wp-content/debug.log for errors

Installation Issues

Problem: Theme Won‚Äôt Install

Cause: File permissions or size limits

Solution:

  1. Check WordPress memory limit:
    • Go to Tools ‚Üí Site Health
    • Look for ‚ÄúMemory limit‚Äù
    • Should be 256MB or higher
  2. Try uploading smaller files first
  3. Use FTP instead of WordPress:
    • Extract theme ZIP
    • Upload folder to /wp-content/themes/
    • Go to Themes, activate SubNews
  4. Contact hosting if still fails

Problem: Theme Appears Corrupted After Installation

Cause: Incomplete download or extraction

Solution:

  1. Delete theme folder via FTP
  2. Re-download theme ZIP
  3. Verify file integrity (check file size)
  4. Extract again
  5. Upload to /wp-content/themes/
  6. Go to Themes, activate SubNews

Problem: Required Plugins Won‚Äôt Install

Cause: Memory limit or conflicts

Solution:

  1. Check memory limit (must be 512MB+ recommended)
  2. Install one plugin at a time
  3. Wait 1 minute between installations
  4. Try installing via FTP:
    • Download plugin ZIP
    • Upload to /wp-content/plugins/
    • Go to Plugins, activate
  5. Contact hosting if needed

Problem: Can‚Äôt Find Theme Options Menu

Cause: Redux Framework not activated

Solution:

  1. Go to Plugins menu
  2. Find ‚ÄúRedux Framework‚Äù
  3. Check it‚Äôs ‚ÄúActive‚Äù
  4. If not, click ‚ÄúActivate‚Äù
  5. Go to Appearance menu
  6. ‚ÄúSubNews Theme Options‚Äù should appear
  7. If not, refresh page

Display Issues

Problem: Website Shows Blank Page

Cause: PHP error or memory limit

Solution:

  1. Enable debug mode (see Before You Start)
  2. Check /wp-content/debug.log for errors
  3. Increase memory limit:
    • Edit wp-config.php
    • Add: define('WP_MEMORY_LIMIT', '512M');
  4. Deactivate all plugins except SubNews and required ones
  5. Switch to default theme, then back to SubNews
  6. Check with hosting provider

Problem: Theme Shows But Colors Are Wrong

Cause: Theme options not saved or conflicting CSS

Solution:

  1. Go to SubNews Theme Options
  2. Click Colors section
  3. Verify colors are set
  4. Click Save Changes
  5. Clear browser cache
  6. Check that custom CSS doesn‚Äôt override colors

Problem: Images Not Displaying

Cause: Wrong file permissions or broken links

Solution:

  1. Check image URL:
    • Go to Media
    • Hover image, get URL
    • Paste in browser address bar
    • Should show image
  2. Fix file permissions:
    • Via FTP, check image files are 644
    • Check folders are 755
  3. Re-upload image:
    • Delete image from Media
    • Upload again
  4. Check image size:
    • Images must be under 5MB
    • Resize if too large

Problem: Header Layout Not Changing

Cause: Cache or CSS issue

Solution:

  1. Go to SubNews Theme Options ‚Üí Header
  2. Choose different layout
  3. Save Changes
  4. Clear browser cache (Ctrl+Shift+Delete)
  5. Wait 30 seconds
  6. Refresh website
  7. If using caching plugin, clear its cache

Problem: Blog Layout Looks Wrong

Cause: CSS issue or conflicting plugin

Solution:

  1. Go to SubNews Theme Options ‚Üí Blog
  2. Choose layout
  3. Save Changes
  4. Clear all caches:
    • Browser cache
    • Plugin cache (if using)
    • Server cache
  5. Disable caching plugins temporarily
  6. Check custom CSS in theme options

Problem: Menu Not Showing Correctly

Cause: Menu not assigned or CSS issue

Solution:

  1. Go to Appearance ‚Üí Menus
  2. Check menu is assigned to location
  3. Edit menu, verify items are added
  4. Check menu items aren‚Äôt password protected
  5. Clear browser cache
  6. Check for CSS conflicts in custom code
  7. Disable plugins that affect menus

Performance Issues

Problem: Website Loading Slowly

Cause: Large images, no caching, too many plugins

Solution Priority:

1. Install Caching Plugin

  • Go to Plugins ‚Üí Add New
  • Search ‚ÄúWP Super Cache‚Äù
  • Install and Activate
  • Settings are automatic

2. Optimize Images

  • Use tool like TinyPNG.com
  • Compress all images before uploading
  • Resize to appropriate dimensions
  • Use JPG for photos, PNG for graphics

3. Disable Unused Plugins

  • Plugins use memory
  • Only keep necessary ones
  • Deactivate unused plugins

4. Use CDN (Optional)

  • Sign up for Cloudflare (free)
  • Changes DNS settings
  • Speeds up delivery globally

5. Clean Database

  • Go to Tools ‚Üí Cleanup Database
  • Or use cleanup plugin
  • Remove old revisions

Problem: Images Loading Slowly

Cause: Large file sizes

Solution:

  1. Compress images before uploading
  2. Use online tool:
    • Visit TinyPNG.com
    • Upload images
    • Download compressed versions
    • Re-upload to WordPress
  3. Delete uncompressed versions
  4. Use lazy loading:
    • Go to SubNews Theme Options
    • Enable lazy loading
    • Save changes

Problem: Page Builder Slow to Load

Cause: Large page with many elements

Solution:

  1. Reduce number of elements on page
  2. Break page into multiple pages
  3. Use simple layouts
  4. Optimize images
  5. Disable extra plugins
  6. Increase memory limit to 512MB

Functionality Issues

Problem: Shortcodes Display as Text

Cause: Plugin not active or syntax error

Solution:

  1. Check Mesh Core plugin is activated
  2. Check shortcode syntax is correct
  3. Try copying shortcode from page builder instead of typing
  4. Clear browser cache
  5. Disable caching plugins
  6. Check for PHP errors (enable debug mode)

Problem: Page Builder Not Opening

Cause: WPBakery not activated or browser issue

Solution:

  1. Check WPBakery is activated
  2. Go to Plugins, find WPBakery
  3. Should show ‚ÄúDeactivate‚Äù link
  4. If shows ‚ÄúActivate‚Äù, click it
  5. Try different browser
  6. Disable browser extensions:
    • AdBlock, uBlock Origin, etc.
    • Disable temporarily
    • Try page builder again
  7. Clear browser cache completely
  8. Restart browser

Problem: Contact Form Not Sending Email

Cause: Mail server or form configuration issue

Solution:

  1. Check email address in form:
    • Go to Contact Forms 7 ‚Üí edit form
    • Click ‚ÄúMail‚Äù tab
    • Verify ‚ÄúTo‚Äù email address
    • Should be your admin email
  2. Test form:
    • Go to page with form
    • Fill it out
    • Submit
    • Check inbox and spam folder
  3. If no email arrives:
    • Check hosting supports email
    • Try different email address
    • Contact hosting provider
  4. Check form isn‚Äôt password protected

Problem: Search Not Working

Cause: Search configuration or plugin conflict

Solution:

  1. Go to SubNews Theme Options ‚Üí Search
  2. Enable search functionality
  3. Save Changes
  4. Test search on website
  5. If still not working:
    • Disable plugins except SubNews + required
    • Test again
    • Enable plugins one by one until finds culprit
  6. Check WordPress posts are published

Problem: Related Posts Not Showing

Cause: Configuration or no related content

Solution:

  1. Go to SubNews Theme Options ‚Üí Blog
  2. Enable ‚ÄúRelated Posts‚Äù
  3. Save Changes
  4. Check post has categories
  5. Check other posts in same category exist
  6. Related posts automatically show on single post
  7. If still not showing:
    • Check post isn‚Äôt password protected
    • Check categories are assigned
    • Clear all caches

Problem: WooCommerce Products Not Showing

Cause: WooCommerce not activated or products not published

Solution:

  1. Check WooCommerce is installed and activated
  2. Check products are published:
    • Go to Products
    • Verify products show
  3. Check products are in category:
    • Edit product
    • Assign category
    • Save
  4. Go to shop page
  5. Products should display
  6. If not:
    • Check theme options WooCommerce settings
    • Clear all caches
    • Disable plugins except SubNews + WooCommerce

Plugin Conflicts

Problem: Two Plugins Conflicting

Cause: Plugin incompatibility or code conflict

Solution:

  1. Identify which plugins conflict (usually 2 plugins)
  2. Deactivate one plugin
  3. Test if issue goes away
  4. If yes, you found culprit
  5. Options:
    • Use different plugin
    • Contact plugin developer
    • Disable one plugin

To Find Conflicting Plugin:

  1. Deactivate all plugins except SubNews + required ones
  2. Test if problem gone
  3. Activate plugins one by one
  4. Test after each activation
  5. When problem reappears, found culprit
  6. Disable that plugin

Problem: Plugin Causes White Screen

Cause: Memory limit or incompatible plugin

Solution:

  1. Increase memory limit:
    • Edit wp-config.php
    • Add: define('WP_MEMORY_LIMIT', '512M');
  2. Deactivate all plugins except SubNews + required
  3. Activate plugins one by one
  4. When problem reappears, found culprit
  5. Delete or replace that plugin

Problem: Plugin Breaks Page Builder

Cause: Plugin modifies core functionality

Solution:

  1. Deactivate all non-essential plugins
  2. Test page builder
  3. Activate plugins one by one
  4. Test after each
  5. When breaks, found culprit
  6. Disable that plugin
  7. Try alternative plugin if available

Security Issues

Problem: Site Hacked or Compromised

Cause: Weak password, outdated software, or malicious plugin

Solution:

  1. Change all passwords immediately:
    • WordPress admin
    • FTP/SFTP
    • cPanel
    • Database
  2. Restore from clean backup (before hack)
  3. Check for malicious plugins:
    • Go to Plugins
    • Look for suspicious plugins
    • Delete if necessary
  4. Update everything:
    • WordPress core
    • All plugins
    • Theme
  5. Install security plugin:
    • Wordfence (free)
    • Sucuri (free)
  6. Scan for malware

Problem: Suspicious Plugin Detected

Cause: Compromised or malicious code

Solution:

  1. Deactivate plugin immediately
  2. Delete plugin
  3. Search plugin name online
  4. Check if it‚Äôs known malware
  5. If legitimate, reinstall fresh copy from WordPress.org
  6. Install security plugin to scan site
  7. Change all passwords

Problem: Brute Force Attack (Many Login Attempts)

Cause: Weak admin password

Solution:

  1. Change admin password to strong one:
    • 12+ characters
    • Mix of letters, numbers, symbols
    • Unique password
  2. Install security plugin:
    • Wordfence
    • iThemes Security
  3. Enable two-factor authentication
  4. Limit login attempts:
    • Wordfence has this built-in
  5. Change login URL:
    • Use security plugin
    • Makes harder to find

Debug Mode Guide

Enable Debug Mode

  1. Edit file: wp-config.php (in root directory)
  2. Find: define('WP_DEBUG', false);
  3. Change to:
    define('WP_DEBUG', true);
define('WP_DEBUG_DISPLAY', false);
define('WP_DEBUG_LOG', true);
  4. Save file

Check Error Log

  1. Via FTP, go to /wp-content/
  2. Open file: debug.log
  3. Look at last entries
  4. Error message will be there

Disable Debug Mode

When done troubleshooting:

  1. Edit wp-config.php
  2. Change back to: define('WP_DEBUG', false);
  3. Delete debug.log file
  4. Save
