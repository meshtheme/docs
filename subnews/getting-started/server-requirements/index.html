Server Requirements for SubNews

SubNews works best on the latest version of WordPress. If you are not on the latest version, make sure you are running WordPress 5.8 or higher.

Minimum requirements (WordPress)

WordPress recommends:

  • PHP version 7.4 or greater.
  • MySQL version 5.6 or greater OR MariaDB version 10.1 or greater.
  • HTTPS support.

For the latest WordPress requirements, see: WordPress Requirements.

Recommended PHP configuration

Low PHP limits can cause issues like demo import failures, slow admin screens, or timeouts. We recommend at least the following:

  • memory_limit 256M
  • max_input_vars 3000
  • max_execution_time 180
  • post_max_size 32M
  • upload_max_filesize 32M
  • allow_url_fopen On

If you are unsure how to check these values, ask your host to confirm or adjust them for you.

Increase the WordPress memory limit

If demo import runs out of memory, raise the WordPress memory limit. You can do this yourself or ask your host to do it for you.

Do It Yourself

To adjust on your own, here are some methods to try. Be aware that this section requires advanced knowledge; it is not basic.

Edit your wp-config.php file

Add this near the top, before the line that says “Happy Blogging”:

define('WP_MEMORY_LIMIT', '256M');

WordPress memory can be different from the server. Set this regardless of server memory settings.

Read More About Editing wp-config.php

Edit your PHP.ini file

If you have access to your php.ini file, update the line below. If it shows 64M, try 256M:

memory_limit = 256M ; Maximum amount of memory a script may consume (64MB)

Edit your .htaccess file

If you don’t have access to php.ini, try adding this to your .htaccess file:

php_value memory_limit 256M

Contact Your Host

If you are not comfortable with the steps above or they do not work, ask your host to increase the memory limit for you.
