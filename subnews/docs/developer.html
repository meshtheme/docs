üìñ Common Tasks

Add Custom Action Hook

Theme Example:

add_action( 'subnews_before_wp_footer', 'my_custom_footer_content', 15 );
function my_custom_footer_content() {
    echo '<div class="my-banner">' . wp_kses_post( $content ) . '</div>';
}

See API Reference - Action Hooks for all available hooks.

Filter Theme Options

Theme Example:

add_filter( 'subnews_navbar_search_placeholder', 'my_custom_placeholder' );
function my_custom_placeholder( $placeholder ) {
    return __( 'Search articles...', 'my-theme' );
}

See API Reference - Filter Hooks for all available filters.

Create Custom Shortcode

Plugin Pattern:

class My_Custom_Element {
    public function __construct() {
        if ( function_exists( 'vc_add_shortcode_param' ) ) {
            add_action( 'vc_before_init', array( $this, 'mapping' ) );
        }
        add_shortcode( 'my_element', array( $this, 'render' ) );
    }

    public function mapping() {
        vc_map( array(
            'name'    => 'My Element',
            'base'    => 'my_element',
            'params'  => array( /* parameters */ ),
        ) );
    }

    public function render( $atts, $content = null ) {
        $atts = shortcode_atts( array(
            'title' => 'Default Title',
        ), $atts );
        return '<div>' . esc_html( $atts['title'] ) . '</div>';
    }
}
new My_Custom_Element();

See Mesh-Core Plugin Documentation - Shortcodes for more details.

Configure View Counting

Plugin Filters:

// Exclude admins from counting
add_filter( 'mtc_view_count_exclude_admins', '__return_true' );

// Set 50% sampling rate for performance
add_filter( 'mtc_sampling_rate', function() { return 50; } );

// Enable AJAX for cache compatibility
add_filter( 'mtc_view_count_cache_support', '__return_true' );

See API Reference - View Count Filters for more options.

üîí Security Best Practices

Input Sanitization

Always sanitize user input:

$text = sanitize_text_field( $_POST['text'] );
$email = sanitize_email( $_POST['email'] );
$url = esc_url( $_POST['url'] );

Output Escaping

Always escape output:

echo esc_html( $text );
echo '<img alt="' . esc_attr( $alt ) . '" />';
echo '<a href="' . esc_url( $url ) . '">Link</a>';
echo wp_kses_post( $html_content );

Nonce Verification

Use nonces for form submissions and AJAX:

// Verify nonce
check_ajax_referer( 'my_nonce', 'nonce_field' );
wp_verify_nonce( $_POST['nonce'], 'my_nonce' );

Capability Checks

Verify user capabilities for sensitive operations:

if ( ! current_user_can( 'manage_options' ) ) {
    wp_die( 'Unauthorized' );
}

See API Reference - Security for more details.

üéØ Architecture Overview

Theme Bootstrap Flow

  1. functions.php loads includes/bootstrap.php
  2. bootstrap.php loads all core classes
  3. Classes instantiate as singletons:
    • SubNews_Assets
    • SubNews_Menus
    • SubNews_Template_Functions
    • SubNews_Style_Generator
  4. Redux options are loaded
  5. Metaboxes are registered
  6. AJAX handlers are registered
  7. Hooks are registered via SubNews_*_Functions classes

Plugin Bootstrap Flow

  1. mesh-core.php loads includes/bootstrap.php
  2. bootstrap.php loads all core classes
  3. Mesh_Core class initializes
  4. Mesh_Core_Loader registers all hooks
  5. Element classes auto-instantiate (register VC + shortcodes)
  6. Widget classes auto-register
  7. Helper functions are loaded

Hook System

üêõ Troubleshooting & Support

Common Issues

Theme not loading assets:

  • Check wp_enqueue_script() priority (should be 110)
  • Verify file paths are correct
  • Check for asset URL generation

Shortcodes not rendering:

  • Verify Visual Composer is active
  • Check element class is being loaded
  • Look for PHP errors in debug log

AJAX requests failing:

  • Verify nonce is correct
  • Check admin-ajax.php is accessible
  • Review browser console for errors

Hooks not firing:

  • Verify hook name is correct
  • Check callback function exists
  • Review priority and accepted_args

Debug Mode

Enable WordPress debug mode:

// wp-config.php
define( 'WP_DEBUG', true );
define( 'WP_DEBUG_LOG', true );
define( 'WP_DEBUG_DISPLAY', false );

Then check /wp-content/debug.log for errors.
