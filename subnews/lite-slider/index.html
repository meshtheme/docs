Overview

The Lite Slider allows you to create beautiful, responsive sliders without any coding. Each slide can have:

  • Custom background image or video
  • Rich text content with HTML formatting
  • Overlay colors for better text readability
  • Two customizable call-to-action buttons
  • Mobile-optimized focal point positioning
  • Flexible text positioning and alignment

Access Lite Slider: Go to SubNews → Lite Slider in your WordPress dashboard.

Creating Slides

Add New Slide

To add a new slide, click the “Add New Slider” button in the Lite Slider settings.

Slide Options

1. Background Color

Default: Transparent (rgba(0,0,0,0))

Set a background color for this slide. Useful when:

  • Using no background image
  • Creating solid color slides
  • Adding color behind semi-transparent images
  • Creating gradient effects (via custom CSS)

Supports: RGB, RGBA (with transparency), HEX colors

2. Background Image

Required field

Upload your slide background image or video.

Supported Formats:

  • Images: JPG, JPEG, PNG, GIF, WebP
  • Videos: MP4, WebM, WebP, OGG, WMV, MOV, AVI

Recommended Image Specifications:

  • Dimensions: 1920x1080px (Full HD) or 2560x1440px (2K)
  • Aspect Ratio: 16:9 for standard screens
  • File Size: Under 500KB (optimize before upload)
  • Format: JPG for photos, PNG for graphics with transparency

Video Background Tips:

  • Keep videos under 10MB for performance
  • Use MP4 format for best compatibility
  • Videos auto-play (muted) on desktop
  • Videos don’t auto-play on mobile (shows poster frame)
  • Compress videos before uploading

3. Mobile/Tablet Image Focal Point

Default: 50% 50% (Center)

Control which part of your image remains visible on smaller screens.

How to Use:

  1. Drag the red-and-white focal point dot on the image preview
  2. Position it on the most important part of your image
  3. Or enter custom position values manually

Position Values:

  • First value: Horizontal position (left to right)
  • Second value: Vertical position (top to bottom)

Examples:

  • 50% 50% - Center (default)
  • 50% 30% - Center horizontally, upper third vertically
  • 20% 50% - Left side, centered vertically
  • 80% 50% - Right side, centered vertically
  • 50% 70% - Center horizontally, lower third vertically

Why This Matters: Large images get cropped on mobile devices. The focal point ensures your subject (person’s face, product, logo) stays visible and centered on all screen sizes.

4. Description

Supports: Full HTML and WordPress editor

Add your slide content using the visual editor:

  • Headlines and subheadings
  • Paragraphs and text
  • Lists (bulleted or numbered)
  • Bold, italic, underline formatting
  • Links
  • Custom HTML

Tips:

  • Keep text concise for better readability
  • Use heading tags (H1-H6) for proper hierarchy
  • Test readability against your background image
  • Use the overlay color to improve text contrast

5. Slide Overlay Color

Default: rgba(0, 0, 0, 0.1) (10% black)

Add a semi-transparent color layer over the background image to improve text readability.

Common Overlay Colors:

  • rgba(0, 0, 0, 0.3) - 30% black (subtle darkening)
  • rgba(0, 0, 0, 0.5) - 50% black (medium darkening)
  • rgba(0, 0, 0, 0.7) - 70% black (strong darkening)
  • rgba(255, 255, 255, 0.3) - 30% white (lightening)
  • rgba(29, 53, 87, 0.6) - 60% dark blue (branded overlay)

Tips:

  • Darker overlays work better with white text
  • Lighter overlays work better with dark text
  • Adjust opacity (last value) to find the right balance
  • Match overlay color to your brand for consistency

6. Slide Text Color

Default: #ffffff (White)

Set the color for all text in this slide.

Best Practices:

  • Use white (#ffffff) with dark backgrounds/overlays
  • Use dark colors (#333333) with light backgrounds
  • Ensure sufficient contrast for readability
  • Test on actual devices for visibility
  • Consider accessibility standards (WCAG AA minimum)

7. Text Position

Default: Center Center

Choose where your text content appears on the slide.

Available Positions:

  • Top Left - Upper left corner
  • Top Center - Top, horizontally centered
  • Top Right - Upper right corner
  • Center Left - Left side, vertically centered
  • Center Center - Dead center (default, most common)
  • Center Right - Right side, vertically centered
  • Bottom Left - Lower left corner
  • Bottom Center - Bottom, horizontally centered
  • Bottom Right - Lower right corner

Usage Tips:

  • Center Center: Best for hero sections, featured content
  • Top positions: Good for banners with action buttons
  • Bottom positions: Works well with people/products in upper portion
  • Side positions: Creates dynamic, magazine-style layouts

8. Text Position Spacing

Default: None

Add additional left or right spacing for extra-large screens (XL breakpoint and above).

Available Options:

Left Spacing:

  • ps-xl-1--ms-xl-1 - Left space 1 (small)
  • ps-xl-2--ms-xl-2 - Left space 2
  • ps-xl-3--ms-xl-3 - Left space 3
  • ps-xl-4--ms-xl-4 - Left space 4
  • ps-xl-5--ms-xl-5 - Left space 5 (large)

Right Spacing:

  • pe-xl-1--me-xl-1 - Right space 1 (small)
  • pe-xl-2--me-xl-2 - Right space 2
  • pe-xl-3--me-xl-3 - Right space 3
  • pe-xl-4--me-xl-4 - Right space 4
  • pe-xl-5--me-xl-5 - Right space 5 (large)

When to Use:

  • Push text away from screen edges on wide monitors
  • Create asymmetric layouts
  • Accommodate specific design requirements
  • Balance with background image composition

Note: Only affects XL screens (1200px and wider). Smaller screens use standard responsive spacing.

9. Text Alignment

Default: Left

Set horizontal text alignment within the text block.

Options:

  • Left - Text aligned to left edge (default, most readable)
  • Center - Text centered (good for short headlines)
  • Right - Text aligned to right edge (creates dynamic look)

Recommendation:

  • Left alignment: Best for body text and paragraphs
  • Center alignment: Good for headlines and short text
  • Right alignment: Use sparingly for design effect

10. Text Area Max-Width

Default: Full width

Limit the maximum width of the text content block.

Example Values:

  • 700px - Narrow column (good for readability)
  • 900px - Medium width
  • 1200px - Wide content
  • 50% - Half the slide width
  • Leave empty for full width

Why Use This: Long lines of text are hard to read. Limiting width to 700-900px improves readability, especially for paragraphs.

Best Practice: For slides with multiple sentences, use 600-800px for optimal reading experience.

11. Primary Button Link

Optional

Enter the URL where the primary button should link.

Supported URL Types:

  • External: https://example.com
  • Internal page: /about-us/
  • Anchor link: #section-id
  • Email: mailto:info@example.com
  • Phone: tel:+1234567890

Tips:

  • Use absolute URLs for external links
  • Test all links after publishing
  • Consider opening external links in new tab (via custom code)

12. Primary Button Text

Optional

Enter the text displayed on the primary button.

Examples:

  • “Learn More”
  • “Get Started”
  • “Shop Now”
  • “View Portfolio”
  • “Contact Us”
  • “Download Free Guide”

Best Practices:

  • Use action-oriented text
  • Keep it short (2-4 words)
  • Make it specific and clear
  • Avoid generic “Click Here”

13. Secondary Button Link

Optional

Enter the URL for the secondary (second) button.

Same format as Primary Button Link.

When to Use Secondary Button:

  • Offer alternative action (e.g., “Learn More” + “Watch Video”)
  • Provide choice (e.g., “Buy Now” + “Try Demo”)
  • Create hierarchy of actions

14. Secondary Button Text

Optional

Text for the secondary button.

Examples:

  • “Watch Video”
  • “Learn More”
  • “See Demo”
  • “View Details”

Tip: Secondary button should represent a less prominent action than primary button.

Slider Control Options

Global settings that apply to the entire slider.

1. Slider Height

Default: 550px

Set the minimum height of all slides.

Common Values:

  • 450px - Compact height for banners
  • 550px - Default, good balance (default)
  • 650px - Tall slider
  • 80vh - 80% of viewport height
  • 100vh - Full screen height (hero slider)

Units Accepted:

  • px - Fixed pixels (e.g., 550px)
  • % - Percentage of parent container
  • vh - Viewport height (e.g., 100vh for full screen)

Recommendations:

  • Hero sliders: 80vh to 100vh
  • Section sliders: 500px to 650px
  • Banner sliders: 350px to 500px

Responsive Behavior: Height automatically adjusts on smaller screens for optimal viewing.

2. Enable Slider Arrow

Default: Show (Enabled)

Show or hide navigation arrows on the sides of the slider.

Options:

  • Show - Display left/right arrow buttons (default)
  • Hide - No arrow navigation

When to Hide:

  • Single slide (no navigation needed)
  • Mobile-only designs
  • Minimal, clean look
  • When using only indicators or autoplay

Arrow Styling: Arrows can be customized via Theme Options → Colors or custom CSS.

3. Enable Slider Indicators (Breadcrumb)

Default: Show (Enabled)

Show or hide slide indicators (dots) at the bottom of the slider.

Options:

  • Show - Display indicator dots (default)
  • Hide - No indicators

Indicators Show:

  • Current slide position
  • Total number of slides
  • Clickable navigation to specific slides

When to Hide:

  • Single slide
  • Minimal design preference
  • When using only arrows

4. Slide Interval Time

Default: 7000 (7 seconds)

Time in milliseconds between automatic slide transitions.

Common Values:

  • 0 - Disable autoplay (manual navigation only)
  • 5000 - 5 seconds (fast)
  • 7000 - 7 seconds (default, recommended)
  • 10000 - 10 seconds (slow, good for text-heavy slides)
  • 15000 - 15 seconds (very slow)

Conversion:

  • 1 second = 1000 milliseconds
  • 5 seconds = 5000
  • 7 seconds = 7000
  • 10 seconds = 10000

Recommendations:

  • Image-focused: 5-7 seconds
  • Text-heavy: 8-12 seconds
  • Videos: Disable autoplay (set to 0)
  • Call-to-action: 7-10 seconds (give time to read and click)

Tip: If users need time to read content, use longer intervals or disable autoplay entirely.

5. Select Slider Animation

Default: Slide

Choose the transition effect between slides.

Animation Options:

Slide (Default)

  • Classic horizontal sliding effect
  • Smooth left-to-right transition
  • Works well with all content types
  • Most familiar to users
  • Best performance

Fade In (Carousel Fade)

  • Cross-fade transition
  • Current slide fades out while next fades in
  • Elegant, sophisticated effect
  • Good for image-heavy sliders
  • Best for high-quality photos
  • Creates seamless transition

Which to Choose:

  • Slide: Dynamic, directional, classic (recommended for most cases)
  • Fade: Elegant, smooth, modern (best for portfolios, photography)

Performance Note: Both animations are optimized and perform well on all devices.

6. Slider Bottom Separator

Default: None

Add a decorative SVG shape separator between the slider and content below.

Available Separator Styles:

Wave Separators:

  • wave1 through wave15 - 15 different wave patterns
  • Smooth, organic shapes
  • Great for modern, flowing designs

Opacity Separators:

  • curve-asym-opacity - Asymmetric curve with gradient
  • fan-opacity - Fan/radial pattern with fade
  • tilt-opacity - Tilted separator with transparency

Geometric Separators:

  • curve - Simple curved separator
  • triangle - Triangular point separator
  • triangle-edge - Sharp triangle edge
  • tilt - Diagonal slanted separator
  • arrow - Arrow-shaped separator
  • book - Book fold effect

Themed Separators:

  • clouds - Cloud shapes
  • iceberg - Iceberg peaks
  • city-skyline - City building silhouettes
  • faded-triangle - Gradient triangle

Creative Separators:

  • torn-paper - Torn paper edge effect
  • music-notes - Musical note decorations
  • paint-drip - Paint dripping effect

Choosing a Separator:

  • Waves: Universal, works with any theme
  • Geometric: Clean, modern designs
  • Themed: Match your content (city for urban, clouds for sky/travel)
  • Creative: Bold, artistic designs

Example Use Cases:

  • Ocean/water theme → wave1-wave15
  • Tech/modern → tilt, triangle
  • Nature/organic → clouds, curve
  • Creative/artistic → paint-drip, torn-paper
  • Music/entertainment → music-notes

7. Separator Height

Default: Empty (uses separator’s default height)

Set the height of the SVG separator in pixels.

Recommended Values:

  • 50 - Subtle, small separator
  • 100 - Medium separator (recommended)
  • 150 - Large separator
  • 200 - Extra large, dramatic effect

Tips:

  • Larger heights create more dramatic visual impact
  • Smaller heights are more subtle
  • Test on mobile - separators scale responsively
  • Adjust based on your overall design

Note: Enter number only (without “px”). Example: 100

8. Separator Background Color

Default: Transparent/None

Set the fill color for the SVG separator.

Common Uses:

  • Match your page background color
  • Match section below slider
  • Use brand colors
  • Create contrast

Examples:

  • #ffffff - White (for white page backgrounds)
  • #f8f9fa - Light gray
  • #1a1a1a - Dark background
  • rgba(0, 0, 0, 0.1) - Semi-transparent

Best Practice: Set this to match the background color of the section directly below your slider for seamless integration.

9. Flip/Inverted Separator

Default: Default (no flip)

Flip or mirror the separator shape.

Options:

  1. Default - Original orientation (no flip)
  2. Flip Vertically - Flip up to down (mirror vertically)
  3. Flip Horizontally - Flip right to left (mirror horizontally)
  4. Flip Vertically & Horizontally - Flip both directions

When to Use:

  • Flip Vertically: Reverse separator direction (point up instead of down)
  • Flip Horizontally: Reverse asymmetric separators for variety
  • Both: Complete 180° rotation for unique effects

Creative Use: Try different flip options to see which looks best with your specific separator and design.

Using the Slider

Inserting the Slider

After creating your slides, use the shortcode to display the slider anywhere on your site.

Shortcode:

[mtc_lite_slider]

Where to Use:

  1. Pages/Posts: Add shortcode directly in content editor
  2. Page Builder: Use shortcode element/widget
  3. Widgets: Use text widget with shortcode
  4. Theme Templates: Use do_shortcode('[mtc_lite_slider]') in PHP
  5. Gutenberg: Use “Shortcode” block

Visual Composer / WPBakery:

  1. Edit page with WPBakery
  2. Click “Add Element”
  3. Select “Raw HTML” or “Shortcode” element
  4. Paste [mtc_lite_slider] shortcode
  5. Save and preview

Best Practices

Image Optimization

Before Uploading:

  1. Resize to 1920x1080px or 2560x1440px
  2. Compress using TinyPNG or similar tool
  3. Target file size: 200-500KB
  4. Use JPG for photos, PNG for graphics

Tools:

  • TinyPNG - Online compression
  • ImageOptim - Mac app
  • Squoosh - Browser-based

Text Readability

Ensure Text is Readable:

  1. Use overlay colors for contrast
  2. Test on actual devices
  3. Limit text length
  4. Use appropriate font sizes
  5. Ensure color contrast (white on dark, dark on light)

Contrast Checker: Use WebAIM Contrast Checker to verify text is readable.

Mobile Optimization

Mobile-Specific Tips:

  1. Set focal points for all images
  2. Test on actual mobile devices
  3. Keep text concise (mobile screens are small)
  4. Use larger button text for touch targets
  5. Consider disabling autoplay on mobile (better UX)
  6. Avoid too many slides (3-5 recommended)

Performance

Keep Sliders Fast:

  1. Limit to 3-7 slides maximum
  2. Optimize all images before upload
  3. Avoid very large video files
  4. Use lazy loading for images
  5. Consider using CDN for media files

Accessibility

Make Sliders Accessible:

  1. Ensure sufficient color contrast
  2. Provide keyboard navigation (built-in)
  3. Add descriptive alt text to images (in media library)
  4. Don’t rely solely on color to convey information
  5. Give users enough time to read (7+ seconds)

Troubleshooting

Slider Not Displaying

Solutions:

  1. Verify shortcode is correct: [mtc_lite_slider]
  2. Check if slides are created and published
  3. Clear all caches (plugin, browser, server)
  4. Disable conflicting plugins temporarily
  5. Check JavaScript console for errors (F12)
  6. Ensure jQuery is loading properly

Images Not Showing

Solutions:

  1. Verify images uploaded successfully
  2. Check file permissions (644 for files, 755 for folders)
  3. Re-upload images if necessary
  4. Check browser console for 404 errors
  5. Verify image URLs are correct
  6. Clear browser cache

Slider Height Issues

Solutions:

  1. Check slider height setting (not 0 or empty)
  2. Use fixed units like px or vh instead of %
  3. Verify parent container has height
  4. Clear cache after changing height
  5. Check for conflicting CSS

Text Not Visible

Solutions:

  1. Increase overlay opacity for better contrast
  2. Change text color to contrast with background
  3. Verify text isn’t positioned off-screen
  4. Check text area max-width isn’t too small
  5. Ensure text color isn’t same as background

Arrows/Indicators Not Showing

Solutions:

  1. Verify arrows/indicators are enabled in settings
  2. Check custom CSS isn’t hiding them
  3. Clear all caches
  4. Verify slider has multiple slides (single slide = no navigation)
  5. Check z-index conflicts with other elements

Autoplay Not Working

Solutions:

  1. Verify interval time is greater than 0
  2. Check JavaScript isn’t blocked
  3. Disable conflicting slider plugins
  4. Check browser console for errors
  5. Some browsers block autoplay (especially with sound)

Separator Not Showing

Solutions:

  1. Select a separator style (not “None”)
  2. Set separator height (try 100px)
  3. Set separator background color
  4. Clear cache after changes
  5. Verify SVG support in browser (all modern browsers support)

Mobile Display Issues

Solutions:

  1. Set focal point for mobile/tablet
  2. Test on actual devices, not just browser resize
  3. Clear mobile browser cache
  4. Check responsive CSS isn’t conflicting
  5. Verify images are responsive (not fixed width)

FAQ

How many slides should I have?

Recommendation: 3-7 slides maximum

Reasoning:

  • Too few (1-2): Not really a slider
  • Sweet spot (3-5): Engaging without overwhelming
  • Too many (10+): Users won’t see them all, hurts performance

Can I have different sliders on different pages?

Currently: The Lite Slider is global - one slider for the entire site.

Workaround: Use conditional shortcodes or page builder logic to display different content, or contact theme support for custom development.

Can I use videos in the slider?

Yes! Upload video files as background images.

Supported formats: MP4, WebM, WebP, OGG, WMV, MOV, AVI

Best practices:

  • Use MP4 for best compatibility
  • Keep under 10MB
  • Videos auto-play muted on desktop
  • Videos don’t auto-play on mobile (performance)

How do I disable autoplay?

Set Slide Interval Time to 0 (zero).

This disables automatic transitions. Slides only change when users click arrows or indicators.

Can I change slider arrow/indicator styles?

Yes! Customize via:

  1. Theme Options → Colors - Change colors
  2. Custom CSS - Full styling control

Example CSS:

/* Change arrow color */
.carousel-control-prev-icon,
.carousel-control-next-icon {
    background-color: #your-color;
}

/* Change indicator color */
.carousel-indicators [data-bs-target] {
    background-color: #your-color;
}

Add custom CSS in Appearance → SubNews Theme Options → Custom Code.

How do I create a full-screen hero slider?

Set Slider Height to 100vh (100% of viewport height).

For best results:

  • Use high-quality images (2560x1440px)
  • Enable transparent header (Page Options metabox)
  • Position text centrally for maximum impact
  • Use strong call-to-action buttons

Can I add custom HTML/CSS to slides?

Yes! The Description field supports full HTML.

Example:

<h1 style="font-size: 60px;">Big Headline</h1>
<p>Subtitle text here</p>
<div class="custom-class">Custom content</div>

Use with caution and test thoroughly.

Why is my slider slow?

Common causes:

  1. Large images - Compress before upload (target: 200-500KB)
  2. Too many slides - Limit to 3-7 slides
  3. Large videos - Keep under 10MB
  4. No caching - Enable caching plugin
  5. Slow hosting - Consider upgrading

See Performance Tips guide for optimization.

Are sliders mobile-friendly?

Yes! The Lite Slider is fully responsive.

Mobile features:

  • Touch swipe navigation
  • Optimized image loading
  • Focal point positioning
  • Adjusted heights
  • Touch-friendly buttons

Always test on actual mobile devices.

Can I schedule slides to show at specific times?

Not built-in, but possible with:

  1. Custom development
  2. Third-party scheduling plugins
  3. Manual show/hide via custom code

Contact theme support for custom solutions.
