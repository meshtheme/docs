SubNews Theme & Mesh-Core Plugin - API Reference

Comprehensive documentation of all public hooks, filters, and extensibility points in the SubNews theme and mesh-core plugin.

Introduction

This reference documents all public hooks (actions and filters) exposed by the SubNews WordPress theme and mesh-core plugin. These hooks enable developers to extend and customize functionality without modifying core files.

Hook Naming Conventions

  • Theme hooks use the subnews_* prefix
  • Plugin hooks use the mesh_* or mtc_* prefixes
  • Content filters use patterns like subnews_the_* or subnews_get_*
  • Configuration filters use patterns like subnews_filter_* or mtc_*

Best Practices

  • Always check hook parameters before using them
  • Use appropriate priorities for multiple callbacks
  • Sanitize and escape all user input
  • Test hooks in a development environment first
  • Document custom implementations for team reference

Theme Filter Hooks

Filter hooks in the SubNews theme allow you to modify and customize data before output.

Theme Options & Settings Filters

subnews_options

Location: plugins/mesh-core/includes/functions/mesh-core-functions.php (line 61) Parameters:

  • $returned (mixed) - The option value
  • $option (string) - The option key

Return Type: Mixed - Filtered option value Purpose: Filter theme options before returning from subnews_opt() function

Usage Example:

add_filter( 'subnews_options', 'my_custom_option_filter', 10, 2 );
function my_custom_option_filter( $value, $option ) {
    if ( 'blog-style' === $option && 'grid' === $value ) {
        return 'custom-grid-layout';
    }
    return $value;
}

Content Filters

Integration Hooks

Shared Filters Between Theme & Plugin

The following filters are shared for cross-compatibility:

  1. mesh_author_bio_avatar_size - Avatar size control
  2. mtc_member_social_links - Team/member social links
  3. Social links container/display - Theme and plugin both respect social link styling

Template Loading Integration

The plugin’s mtc_get_template_part() function uses performance hooks:

do_action( 'start_operation', 'mtc_get_template_part::' . $handle );
// Template part loads from:
// 1. Child theme (if exists)
// 2. Parent theme
// 3. Plugin fallback
do_action( 'end_operation', 'mtc_get_template_part::' . $handle );

Hook Interoperability

  • Both theme and plugin use subnews_opt() for theme options
  • Plugin respects theme settings for styling
  • Theme respects plugin shortcode output
  • Visual Composer integration is seamless between theme and plugin elements

Usage Examples

Example 1: Add Custom Content to Footer

add_action( 'subnews_before_wp_footer', 'my_footer_banner', 25 );
function my_footer_banner() {
    echo '<div class="custom-footer-banner">';
    echo '<p>' . wp_kses_post( 'Custom footer content' ) . '</p>';

Example 2: Customize Search Placeholder

add_filter( 'subnews_navbar_search_placeholder', 'my_search_placeholder' );
function my_search_placeholder( $placeholder ) {
    if ( is_archive() ) {
        return 'Search articles...';
    }
    if ( function_exists( 'is_shop' ) && is_shop() ) {
        return 'Search products...';
    }
    return 'What are you looking for?';
}

Example 3: Modify Category Display

add_filter( 'subnews_the_category', 'my_category_badge', 10, 2 );
function my_category_badge( $html, $type ) {
    if ( 'single' === $type ) {
        return '<span class="category-badge">' . wp_kses_post( $html ) . '</span>';
    }
    return $html;
}

// Exclude admins from view counting

add_filter( 'mtc_sampling_rate', function() {
    return 50;
} );

// Use AJAX for cache compatibility

Example 7: Track Performance

add_action( 'start_operation', 'my_perf_start' );
function my_perf_start( $handle ) {
    global $my_perf;
    $my_perf[ $handle ] = microtime( true );
}

add_action( 'end_operation', 'my_perf_end' );
function my_perf_end( $handle ) {
    global $my_perf;
    if ( isset( $my_perf[ $handle ] ) ) {
        $time = microtime( true ) - $my_perf[ $handle ];
        error_log( $handle . ' took ' . $time . ' seconds' );
    }
}

Best Practices

Security

  • Always sanitize user input with sanitize_*() functions
  • Escape output with esc_html(), esc_attr(), esc_url()
  • Use wp_kses_post() for rich text content
  • Verify nonces for AJAX requests
  • Check capabilities with current_user_can()

Performance

  • Use appropriate hook priorities
  • Avoid heavy operations in frequently called hooks
  • Cache results when possible
  • Use transients for external API calls
  • Consider sampling rates for high-frequency hooks

Compatibility

  • Test with both theme and plugin active
  • Check for conflicts with other plugins
  • Verify WordPress version compatibility
  • Test with multiple browsers
  • Validate with WP_DEBUG enabled

Documentation

  • Document custom hook usage
  • Include code examples
  • Provide clear function descriptions
  • Note any breaking changes
  • Maintain a changelog
