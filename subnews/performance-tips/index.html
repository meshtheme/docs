Why Performance Matters

Website performance directly impacts:

  • User Experience - Faster sites keep visitors engaged
  • SEO Rankings - Google prioritizes fast-loading sites
  • Conversion Rates - Speed increases sales and signups
  • Mobile Users - Critical for mobile network performance
  • Bounce Rates - Slow sites lose visitors quickly

Goal: Achieve load times under 3 seconds on standard connections.

  2. Test from multiple locations
  3. Test both mobile and desktop
  4. Run tests 3 times, average results
  5. Test homepage and key pages
  6. Compare before/after optimization

Image Optimization

Images typically account for 50-70% of page weight. Proper optimization is crucial.

Before Uploading Images

Resize Images

Recommended Maximum Dimensions:
- Full-width images: 1920px wide
- Featured images: 1200px wide
- Thumbnails: 400px wide
- Retina displays: 2x size (but compressed)

Compress Images

Use these tools before uploading:

  • TinyPNG/TinyJPG - https://tinypng.com/
  • ImageOptim (Mac) - https://imageoptim.com/
  • Squoosh (Browser) - https://squoosh.app/
  • ShortPixel (Bulk) - https://shortpixel.com/

Target Compression:

  • JPG photos: 60-80% quality
  • PNG graphics: Optimize with tools
  • Keep file sizes under 200KB when possible

Choose Correct Image Format

FormatBest ForProsCons
JPGPhotos, complex imagesSmall file size, good qualityNo transparency
PNGLogos, graphics, textTransparency support, losslessLarger file size
WebPModern browsers25-35% smaller than JPG/PNGLimited old browser support
SVGIcons, logos, simple graphicsInfinitely scalable, tiny sizeNot for photos

Caching Solutions

Caching stores static versions of your pages to serve quickly.

Types of Caching

1. Browser Caching Stores assets in visitor’s browser for repeat visits.

2. Page Caching Stores entire HTML pages to avoid database queries.

3. Object Caching Caches database queries and API calls.

4. Opcode Caching Caches compiled PHP code (usually server-side).

Content Delivery Network (CDN)

CDN distributes your static files globally for faster delivery worldwide.

What is a CDN?

A CDN stores copies of your static files (images, CSS, JavaScript) on servers worldwide. When someone visits your site, files load from the nearest server.

Benefits:

  • Faster page loads globally
  • Reduced server load
  • Better handling of traffic spikes
  • Improved reliability
  • Enhanced security (DDoS protection)

Recommended CDN Services

Cloudflare (Recommended)

CDN Setup with SubNews

Using Cloudflare (Most Popular):

  1. Sign up and add site:
    • Go to cloudflare.com
    • Add your domain
    • Cloudflare scans DNS records
  2. Update nameservers:
    • Copy Cloudflare nameservers
    • Update at domain registrar
    • Wait for DNS propagation (24-48 hours)
  3. Configure settings:
    • SSL/TLS: Set to “Full” or “Full (Strict)”
    • Speed → Auto Minify: Enable CSS, JS, HTML
    • Speed → Rocket Loader: Enable (test first)
    • Caching → Always Online: Enable
  4. Integrate with WordPress:
    • Install Cloudflare plugin
    • Enter API key
    • Configure caching rules
    • Test site thoroughly

Using Traditional CDN (BunnyCDN example):

  1. Sign up for CDN service
  2. Create Pull Zone (points to your domain)
  3. Copy CDN URL provided
  4. Install CDN Enabler plugin
  5. Enter CDN URL in plugin settings
  6. Test that files load from CDN

Database Optimization

Over time, WordPress databases accumulate unnecessary data.

What Slows Down Database

  • Post revisions (unlimited drafts)
  • Spam comments
  • Trashed content
  • Transients (expired temporary data)
  • Unused tables from old plugins
  • Auto-drafts
  • Orphaned metadata

Manual Database Optimization

Via phpMyAdmin:

  1. Log into cPanel/hosting panel
  2. Open phpMyAdmin
  3. Select WordPress database
  4. Check all tables
  5. Select “Optimize table” from dropdown
  6. Run optimization

Via WP-CLI (Advanced):

Database Best Practices

Limit Post Revisions:

Add to wp-config.php:

// Limit revisions to 3
define('WP_POST_REVISIONS', 3);

Disable Revisions (not recommended):

define('WP_POST_REVISIONS', false);

Regular Maintenance:

  • Optimize database weekly/monthly
  • Delete spam comments regularly
  • Empty trash frequently
  • Remove unused plugins completely
  • Clean transients periodically

Minification & Compression

Reduce file sizes by removing unnecessary characters and compressing files.

What is Minification?

Minification removes:

  • Whitespace and line breaks
  • Comments
  • Unused code
  • Formatting characters

Example:

/* Before Minification (1.2 KB) */
.header {
    background-color: #ffffff;
    padding: 20px;
    margin: 0 auto;
}

/* After Minification (0.6 KB) */
.header{background-color:#fff;padding:20px;margin:0 auto}

Minification Methods

Method 1: WP Rocket (Easiest)

  • Install WP Rocket
  • Go to File Optimization
  • Enable “Minify CSS files”
  • Enable “Combine CSS files”
  • Enable “Minify JavaScript files”
  • Enable “Combine JavaScript files”
  • Test site thoroughly

Method 2: Autoptimize (Free)

  • Install Autoptimize plugin
  • Go to Settings → Autoptimize
  • Enable “Optimize HTML Code”
  • Enable “Optimize CSS Code”
  • Enable “Optimize JavaScript Code”
  • Save and clear cache

Method 3: W3 Total Cache

  • Go to Performance → Minify
  • Enable minification
  • Select “Manual” mode
  • Add CSS files to minify
  • Add JS files to minify
  • Test carefully

Method 4: Cloudflare (Server-side)

  • Cloudflare dashboard
  • Speed → Optimization
  • Enable “Auto Minify” for CSS, JS, HTML
  • No plugin needed

GZIP Compression

GZIP reduces file sizes by 50-70% during transfer.

Check if GZIP is Enabled:

  • Use Check GZIP Compression
  • Enter your URL
  • Check results

Enable GZIP:

Via Plugin (Easiest):

  • WP Rocket enables automatically
  • W3 Total Cache: Performance → Browser Cache → Enable

Via .htaccess (Manual): Add to .htaccess:

<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html
  AddOutputFilterByType DEFLATE text/css
  AddOutputFilterByType DEFLATE text/javascript
  AddOutputFilterByType DEFLATE text/xml
  AddOutputFilterByType DEFLATE application/javascript
  AddOutputFilterByType DEFLATE application/x-javascript
</IfModule>

Via Cloudflare: Automatic GZIP compression for all files.

✅ Do:

  • Check all functionality

❌ Don’t:

  • Combine all scripts blindly

Reduce HTTP Requests

Each file (image, CSS, JavaScript) requires a separate HTTP request. Fewer requests = faster loading.

Common HTTP Request Sources

  • Images (largest contributor)
  • CSS files
  • JavaScript files
  • Fonts
  • External scripts (ads, analytics)
  • Social media widgets

Reduction Strategies

1. Combine Files

  • Combine multiple CSS files into one
  • Combine JavaScript files
  • Use CSS sprites for icons
  • Use icon fonts instead of images

2. Remove Unnecessary Scripts

Common Unnecessary Scripts:
- Unused social sharing buttons
- Multiple analytics codes
- Unused plugin scripts
- Unnecessary fonts
- Auto-loaded embeds

3. Load Scripts Conditionally Only load scripts where needed:

  • Contact form scripts only on contact page
  • Slider scripts only on pages with sliders
  • Gallery scripts only on gallery pages

4. Use Plugin Organizer Control which plugins load on which pages:

  1. Install Plugin Organizer
  2. Disable plugins per page/post type
  3. Reduce unnecessary scripts
  4. Improve page speed

5. Limit External Scripts

  • Use local analytics (self-hosted)
  • Minimize social widgets
  • Avoid auto-loading embeds
  • Use async/defer for external scripts

Monitor HTTP Requests

Using Browser DevTools:

  1. Open site in Chrome/Firefox
  2. Press F12 (Developer Tools)
  3. Go to Network tab
  4. Refresh page
  5. Check number of requests
  6. Identify heavy files

Target: Under 50 HTTP requests per page (fewer is better).

Server Requirements

Minimum Requirements:

  • PHP 7.4 or higher (8.0+ recommended)
  • MySQL 5.6 or higher
  • 256 MB memory limit (512 MB recommended)
  • Apache or Nginx server
  • HTTPS/SSL certificate

Optimal Requirements:

  • PHP 8.1+
  • MySQL 8.0+
  • 512 MB+ memory limit
  • LiteSpeed server (fastest)
  • HTTP/2 support
  • SSL certificate

Server-Level Optimizations

Enable OPcache (PHP): Ask host to enable PHP OPcache for faster PHP execution.

Use LiteSpeed Server: LiteSpeed is significantly faster than Apache. Some hosts offer it.

Enable HTTP/2: HTTP/2 loads multiple files simultaneously. Most modern hosts support it.

Use PHP 8.0+: PHP 8 is 2-3x faster than PHP 7.4. Update if available.

Plugin Management

Plugins can significantly slow down your site.

Plugin Impact

Each active plugin can:

  • Add HTTP requests
  • Load CSS/JavaScript files
  • Make database queries
  • Increase page generation time
  • Introduce compatibility issues

Plugin Best Practices

Audit Your Plugins:

  1. List all active plugins
  2. Test disabling each one
  3. Measure speed impact
  4. Keep only essential plugins
  5. Find lighter alternatives

How Many Plugins?

  • Quality over quantity
  • 10-20 well-coded plugins is fine
  • 50+ poorly-coded plugins is problematic
  • Monitor performance per plugin

Choose Quality Plugins:

  • Check reviews and ratings
  • Verify regular updates
  • Test performance impact
  • Check compatibility
  • Read support forum

Lazy Load Plugins:

  • Use Plugin Organizer
  • Load plugins conditionally
  • Disable on specific pages
  • Reduce global scripts

Essential vs. Optional Plugins

Essential (Keep):

  • Security (Wordfence, iThemes)
  • Backup (UpdraftPlus)
  • SEO (Yoast, Rank Math)
  • Performance (WP Rocket)
  • Contact Form (Contact Form 7)

Often Unnecessary:

  • Related posts (SubNews has built-in)
  • Social sharing (use simple code)
  • “All-in-one” plugins (use specific tools)
  • Slider plugins (use Lite Slider)
  • Multiple SEO plugins (use one!)

WordPress Optimization

Core WordPress optimizations for better performance.

Disable Unused Features

Disable Embeds: Add to functions.php (or child theme):

// Disable embeds
function disable_embeds_code_init() {
    remove_action('rest_api_init', 'wp_oembed_register_route');
    remove_filter('oembed_dataparse', 'wp_filter_oembed_result', 10);
    remove_action('wp_head', 'wp_oembed_add_discovery_links');
    remove_action('wp_head', 'wp_oembed_add_host_js');
}
add_action('init', 'disable_embeds_code_init', 9999);

Disable Emojis:

// Disable emojis
function disable_emojis() {
    remove_action('wp_head', 'print_emoji_detection_script', 7);
    remove_action('admin_print_scripts', 'print_emoji_detection_script');
    remove_action('wp_print_styles', 'print_emoji_styles');
    remove_action('admin_print_styles', 'print_emoji_styles');
}
add_action('init', 'disable_emojis');

Disable XML-RPC:

// Disable XML-RPC
add_filter('xmlrpc_enabled', '__return_false');

Limit Post Revisions

Add to wp-config.php:

define('WP_POST_REVISIONS', 3);

Increase Memory Limit

Add to wp-config.php:

define('WP_MEMORY_LIMIT', '512M');

Disable Pingbacks/Trackbacks

  1. Go to Settings → Discussion
  2. Uncheck “Allow link notifications from other blogs”
  3. Save changes

Update WordPress Regularly

  • Keep WordPress core updated
  • Update plugins regularly
  • Update themes regularly
  • Test updates on staging first

Advanced Optimizations

Advanced techniques for maximum performance.

Object Caching (Redis/Memcached)

What it does: Caches database queries and objects in memory for instant access.

Requirements:

  • VPS or dedicated hosting
  • Redis or Memcached server
  • Technical knowledge or host support

Setup (Redis):

  1. Install Redis on server
  2. Install Redis Object Cache plugin
  3. Activate plugin
  4. Go to Settings → Redis
  5. Enable object cache
  6. Monitor performance

Performance Gain: 20-50% faster page generation

Preload Critical Assets

Preload Fonts: Add to theme header:

<link rel="preload" href="/fonts/font-name.woff2" as="font" type="font/woff2" crossorigin>

Preload Key CSS:

<link rel="preload" href="/css/critical.css" as="style">

Defer JavaScript Loading

Defer Non-Critical JS:

<script src="script.js" defer></script>

Async for External Scripts:

<script src="analytics.js" async></script>

Critical CSS

Extract critical above-the-fold CSS and inline it.

Tools:

  • Critical CSS Generator: https://jonassebastianohlsson.com/criticalpathcssgenerator/
  • WP Rocket (automatic critical CSS)
  • Autoptimize (critical CSS feature)

Prefetch DNS

Preconnect to external domains:

<link rel="dns-prefetch" href="//fonts.googleapis.com">
<link rel="dns-prefetch" href="//ajax.googleapis.com">

Service Workers (PWA)

Implement Progressive Web App features:

  • Install SuperPWA plugin
  • Configure offline caching
  • Enable push notifications
  • Improve repeat visits

Core Web Vitals

Google’s Core Web Vitals measure user experience:

LCP (Largest Contentful Paint):

  • Target: Under 2.5 seconds
  • Measures: Main content loading time
  • Improve: Optimize images, use CDN, improve server response

FID (First Input Delay):

  • Target: Under 100 milliseconds
  • Measures: Interactivity responsiveness
  • Improve: Reduce JavaScript, defer scripts

CLS (Cumulative Layout Shift):

  • Target: Under 0.1
  • Measures: Visual stability
  • Improve: Set image dimensions, avoid dynamic content insertion

Troubleshooting Performance Issues

Common Performance Problems

Problem: Site suddenly slow

Solutions:

  1. Check for recent plugin updates
  2. Deactivate recently installed plugins
  3. Clear all caches
  4. Check for brute force attacks
  5. Review error logs
  6. Contact hosting support

Problem: Admin dashboard slow

Solutions:

  1. Increase PHP memory limit
  2. Reduce post revisions
  3. Optimize database
  4. Disable unnecessary admin plugins
  5. Check for theme conflicts

Problem: Slow after update

Solutions:

  1. Clear all caches
  2. Regenerate thumbnails
  3. Deactivate/reactivate plugins
  4. Check for conflicts
  5. Roll back if necessary

Problem: High server load

Solutions:

  1. Check for attacks/spam
  2. Review resource-heavy plugins
  3. Implement caching
  4. Upgrade hosting
  5. Use CDN

Performance Testing After Changes

Always test after making changes:

  1. Clear all caches (plugin, browser, server)
  2. Test in incognito/private window
  3. Run GTmetrix test
  4. Run PageSpeed Insights
  5. Test all functionality
  6. Check mobile performance
  7. Monitor for 24 hours
