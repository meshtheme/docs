Why Security Matters

WordPress powers 43% of all websites, making it a prime target for attackers.

Security Risks

Without proper security, your site faces:

  • Data Breaches - Stolen customer information
  • Malware Infections - Malicious code injection
  • SEO Spam - Hidden links damage rankings
  • Defacement - Site content replaced
  • Downtime - Loss of revenue and reputation
  • Blacklisting - Removed from Google search results
  • Legal Issues - GDPR violations, liability claims

Security Benefits

A secure website provides:

  • Protection for visitor data
  • Maintained SEO rankings
  • Brand reputation protection
  • Customer trust and confidence
  • Business continuity
  • Compliance with regulations
  • Peace of mind

Remember: Prevention is easier and cheaper than recovery.

Common Security Threats

Understanding threats helps prevent them.

1. Brute Force Attacks

What it is: Automated attempts to guess login credentials by trying thousands of username/password combinations.

Signs:

  • Multiple failed login attempts
  • Slow admin dashboard
  • High server load
  • Login lockout messages

Prevention:

  • Strong passwords (see Password Security section)
  • Limit login attempts
  • Two-factor authentication
  • Change default admin username
  • Use CAPTCHA on login page

2. Malware Infections

What it is: Malicious code injected into your site to steal data, redirect visitors, or send spam.

Signs:

  • Unexpected redirects
  • Unknown admin users
  • Modified core files
  • Spam content appearing
  • Google blacklist warning

Prevention:

  • Keep WordPress updated
  • Use security plugins
  • Regular malware scans
  • File integrity monitoring
  • Secure file permissions

3. SQL Injection

What it is: Attackers insert malicious SQL code into input fields to access or manipulate your database.

Signs:

  • Unauthorized database changes
  • Exposed user data
  • Site defacement
  • Unexpected admin users

Prevention:

  • Keep WordPress updated (WordPress prevents this)
  • Use prepared statements in custom code
  • Input validation and sanitization
  • Web Application Firewall (WAF)
  • Security plugins with SQL injection protection

4. Cross-Site Scripting (XSS)

What it is: Malicious scripts injected into web pages viewed by other users.

Signs:

  • Unwanted JavaScript execution
  • Cookie theft
  • Session hijacking
  • Unexpected pop-ups

Prevention:

  • Keep WordPress and plugins updated
  • Input sanitization
  • Output escaping (SubNews does this properly)
  • Content Security Policy headers
  • Security plugins

5. DDoS Attacks

What it is: Overwhelming your server with traffic to make your site unavailable.

Signs:

  • Site extremely slow or down
  • High server resource usage
  • Increased traffic from suspicious sources
  • Multiple requests from same IPs

Prevention:

  • Use Cloudflare (free DDoS protection)
  • Server-level firewalls
  • Rate limiting
  • CDN service
  • Quality hosting provider

6. Phishing & Social Engineering

What it is: Tricking users into revealing credentials or sensitive information.

Signs:

  • Fake login emails
  • Suspicious password reset requests
  • Unusual admin access
  • Social media account compromise

Prevention:

  • User education and training
  • Verify email authenticity
  • Never share login credentials
  • Use official WordPress.org resources only
  • Enable two-factor authentication

7. File Upload Vulnerabilities

What it is: Malicious files uploaded through forms, plugins, or media library.

Signs:

  • Unknown files in uploads folder
  • PHP files in image directories
  • Unexpected file types
  • Backdoor access

Prevention:

  • Restrict file upload types
  • Use security plugins
  • Regular file scans
  • Proper file permissions
  • Disable PHP execution in uploads folder

8. Outdated Software

What it is: Running old versions of WordPress, plugins, or themes with known vulnerabilities.

Signs:

  • Update notifications
  • Security plugin warnings
  • Vulnerability scan alerts
  • Compatibility issues

Prevention:

  • Enable automatic WordPress updates
  • Update plugins immediately
  • Update themes regularly
  • Remove unused plugins/themes
  • Test updates on staging site first

SSL/HTTPS Setup

SSL encrypts data between your site and visitors. Essential for security and SEO.

Why SSL Matters

Benefits:

  • Encrypts sensitive data (passwords, payments)
  • Google ranking boost
  • Browser security indicators
  • Customer trust
  • Required for modern features
  • PCI compliance for e-commerce

Without SSL:

  • “Not Secure” warning in browsers
  • SEO penalties
  • Payment processing blocked
  • Customer distrust
  • Vulnerability to attacks

Getting SSL Certificate

Method 1: Free SSL (Let’s Encrypt)

Most hosts provide free SSL certificates:

Via cPanel:

  1. Log into cPanel
  2. Find “SSL/TLS Status”
  3. Click “Run AutoSSL”
  4. Wait for certificate installation
  5. Force HTTPS (see below)

Via Hosting Panel:

  1. Log into hosting dashboard
  2. Find SSL section
  3. Click “Install Free SSL” or “Let’s Encrypt”
  4. Select domain
  5. Install certificate

Via Cloudflare:

  1. Add site to Cloudflare
  2. Change nameservers
  3. Go to SSL/TLS tab
  4. Select “Full” or “Full (Strict)”
  5. Enable “Always Use HTTPS”

Method 2: Premium SSL

Purchase from:

  • SSL.com
  • DigiCert
  • GoDaddy
  • Namecheap

Cost: $10-100/year

When needed:

  • E-commerce sites (higher trust)
  • Extended validation (EV SSL)
  • Organization validation (OV SSL)
  • Wildcard certificates (multiple subdomains)

Force HTTPS

After installing SSL, force all traffic to HTTPS:

Method 1: Really Simple SSL Plugin (Easiest)

  1. Install “Really Simple SSL” plugin
  2. Activate plugin
  3. Click “Go ahead, activate SSL!”
  4. Done! Plugin handles everything

Method 2: Manual .htaccess (Recommended)

Add to top of .htaccess:

# Force HTTPS
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
</IfModule>

Method 3: WordPress Settings

  1. Go to Settings → General
  2. Change “WordPress Address” to https://
  3. Change “Site Address” to https://
  4. Save changes
  5. Add .htaccess redirect above

SSL Troubleshooting

Problem: Mixed Content Warnings

Cause: Some resources load over HTTP instead of HTTPS.

Solution:

  1. Use Really Simple SSL plugin (auto-fixes)
  2. Or search/replace URLs in database:
    UPDATE wp_posts SET post_content =
REPLACE(post_content, 'http://yourdomain.com', 'https://yourdomain.com');
  3. Use “Better Search Replace” plugin (safer)

Problem: Redirect Loop

Solutions:

  1. Disable Really Simple SSL
  2. Clear browser cache completely
  3. Clear server cache
  4. Check .htaccess rules
  5. Disable conflicting plugins
  6. Contact hosting support

Problem: Certificate Not Trusted

Solutions:

  1. Verify certificate is properly installed
  2. Check certificate is not expired
  3. Ensure certificate matches domain
  4. Clear browser cache
  5. Test in incognito window

Password Security

Weak passwords are the #1 security vulnerability.

Strong Password Guidelines

Password Requirements:

  • Minimum 12 characters (15+ recommended)
  • Mix uppercase and lowercase
  • Include numbers
  • Include symbols (!@#$%^&*)
  • Avoid dictionary words
  • Don’t reuse passwords
  • Never share passwords

Examples:

❌ Weak Passwords:

  • password123
  • admin
  • yourdomain2024
  • qwerty
  • 123456

✅ Strong Passwords:

  • Tr0p!c@l#R@in$2024
  • M00nL!ght&St@rs#99
  • Blu3$ky*Pr!sm@2024
  • D!git@l#Oce@n$77

Password Best Practices

✅ Do:

  • Use password manager
  • Generate random passwords
  • Use unique password per site
  • Change passwords after breach
  • Enable two-factor authentication
  • Use biometric unlock on mobile

❌ Don’t:

  • Write passwords down
  • Share passwords via email
  • Use same password everywhere
  • Use personal information
  • Use keyboard patterns (qwerty)
  • Store in browser (insecure)

Two-Factor Authentication (2FA)

Add extra security layer requiring second verification method.

What is Two-Factor Authentication?

How it works:

  1. Enter username and password (something you know)
  2. Enter code from phone/app (something you have)
  3. Access granted

Benefits:

  • Blocks 99.9% of automated attacks
  • Protects against stolen passwords
  • Easy to implement
  • Industry standard security

2FA Methods

1. Authenticator Apps (Recommended)

  • Google Authenticator
  • Microsoft Authenticator
  • Authy
  • 1Password (built-in)

2. SMS Codes

  • Less secure (SIM swapping risk)
  • Better than nothing
  • Good for backup method

3. Email Codes

  • Least secure option
  • Acceptable for low-risk sites
  • Always have backup method

4. Hardware Keys

  • Most secure option
  • YubiKey, Titan Security Key
  • Expensive but unbreakable

Setting Up 2FA

Method 1: Wordfence (Recommended)

  1. Install Wordfence Security
  2. Go to Wordfence → Login Security
  3. Click Enable 2FA for your account
  4. Download authenticator app (Google Authenticator)
  5. Scan QR code with app
  6. Enter verification code
  7. Save recovery codes
  8. 2FA enabled!

Require 2FA for all admins:

  1. Go to Wordfence → Login Security → Settings
  2. Enable “Require 2FA for administrators”
  3. Set grace period (7 days)
  4. All admins must enable 2FA

Method 2: Two Factor Authentication Plugin

  1. Install “Two Factor Authentication” plugin
  2. Go to Users → Your Profile
  3. Scroll to “Two-Factor Options”
  4. Click “Enable Two-Factor”
  5. Scan QR code with authenticator app
  6. Enter code to verify
  7. Save backup codes

Method 3: Google Authenticator Plugin

  1. Install “Google Authenticator - WordPress Two Factor Authentication”
  2. Go to plugin settings
  3. Click “Configure”
  4. Select authentication method
  5. Scan QR code
  6. Verify with code
  7. Enable for your account

Backup Codes

Always save backup codes!

What they are: Single-use codes for accessing account if you lose phone.

Storage:

  • Store in password manager
  • Print and store securely
  • Never store on phone
  • Don’t share with anyone

If lost:

  • Use backup code to log in
  • Generate new backup codes
  • Reset 2FA
  • Set up 2FA again

2FA Recovery

Lost Phone:

  1. Use backup code to log in
  2. Disable 2FA temporarily
  3. Set up 2FA on new device
  4. Generate new backup codes

Lost Backup Codes:

  1. Log in while you still can
  2. Generate new backup codes
  3. Store securely

Completely Locked Out:

  1. Contact hosting provider
  2. Verify ownership
  3. Access via FTP/phpMyAdmin
  4. Disable 2FA plugin manually
  5. Delete plugin folder or deactivate via database

Via FTP:

  1. Connect via FTP
  2. Navigate to /wp-content/plugins/
  3. Rename 2FA plugin folder
  4. Log in to WordPress
  5. Set up 2FA again

Backup Strategies

Regular backups are your safety net.

Why Backups Matter

Backups protect against:

  • Hacking and malware
  • Server failures
  • Human errors
  • Plugin conflicts
  • Update failures
  • Data corruption
  • Natural disasters

Without backups:

  • Permanent data loss
  • Impossible recovery
  • Rebuild from scratch
  • Loss of business
  • Customer trust damaged

What to Back Up

Complete Backup Includes:

  1. Database
    • All posts and pages
    • Comments
    • User accounts
    • Plugin settings
    • Theme options
  2. Files
    • WordPress core files
    • Themes (custom modifications)
    • Plugins
    • Uploads (images, media)
    • Custom code
  3. Configuration
    • wp-config.php
    • .htaccess
    • Custom configurations

Recommended Schedule:

E-commerceDaily + before changes

Critical Times:

  • Before WordPress updates
  • Before plugin updates
  • Before theme updates
  • Before major changes
  • Before server migrations

WordPress Hardening

Make WordPress more secure by limiting access and removing vulnerabilities.

Disable File Editing

Prevents hackers from editing theme/plugin files from dashboard.

Add to wp-config.php:

// Disable file editor
define('DISALLOW_FILE_EDIT', true);

Disable Plugin/Theme Installation

Prevents unauthorized plugin installations.

Add to wp-config.php:

// Disable plugin/theme installation
define('DISALLOW_FILE_MODS', true);

Change Database Table Prefix

Default wp_ prefix makes SQL injection easier.

During Installation: Change prefix to something unique like sn47_ or xyz_

After Installation (Advanced): Use “Change Table Prefix” plugin or manual database changes (risky).

Hide WordPress Version

Remove version info from source code.

Add to functions.php (child theme):

// Remove WordPress version
remove_action('wp_head', 'wp_generator');

// Remove version from RSS
add_filter('the_generator', '__return_empty_string');

Disable XML-RPC

XML-RPC can be exploited for brute force attacks.

Via Plugin: Install “Disable XML-RPC” plugin.

Via .htaccess:

# Block XML-RPC
<Files xmlrpc.php>
  order deny,allow
  deny from all
</Files>

Protect wp-config.php

Most important WordPress file - contains database credentials.

Add to .htaccess:

# Protect wp-config.php
<files wp-config.php>
  order allow,deny
  deny from all
</files>

Disable Directory Browsing

Prevent listing of directory contents.

Add to .htaccess:

# Disable directory browsing
Options -Indexes

Limit Login Attempts

Prevent brute force attacks.

Via Plugin: Install “Limit Login Attempts Reloaded”

Settings:

  • 5 attempts allowed

Disable Author Archives

Prevents username enumeration.

Add to functions.php:

// Disable author archives
add_action('template_redirect', function() {
    if (is_author()) {
        wp_redirect(home_url());
        exit;
    }
});

Security Headers

Add security headers via .htaccess:

# Security Headers
<IfModule mod_headers.c>
  # Prevent clickjacking
  Header set X-Frame-Options "SAMEORIGIN"

  # Prevent MIME type sniffing
  Header set X-Content-Type-Options "nosniff"

  # Enable XSS protection
  Header set X-XSS-Protection "1; mode=block"

  # Referrer policy
  Header set Referrer-Policy "strict-origin-when-cross-origin"

  # Content Security Policy (adjust as needed)
  Header set Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';"
</IfModule>

File Permissions

Correct permissions:

  • Directories: 755 or 750
  • Files: 644 or 640
  • wp-config.php: 440 or 400

Via FTP:

  1. Right-click file/folder
  2. Choose “File Permissions”
  3. Set correct permissions
  4. Apply to subdirectories if needed

Via SSH:

# Set directory permissions
find /path/to/wordpress/ -type d -exec chmod 755 {} \;

# Set file permissions
find /path/to/wordpress/ -type f -exec chmod 644 {} \;

# Secure wp-config.php
chmod 440 wp-config.php

Firewall blocks:

  • SQL injection attempts
  • XSS attacks
  • Brute force attacks
  • DDoS attacks
  • Malicious bots
  • Known vulnerability exploits
  • Suspicious IP addresses

Firewall types:

  1. Cloud-based (Cloudflare, Sucuri) - Best
  2. Plugin-based (Wordfence) - Good
  3. Server-level (mod_security) - Best but complex
